## How to Contribute

### Reporting Bugs

- Check if the bug has already been reported in [Issues](https://github.com/sawaipratap/amcatbuddy/issues)
- If not, create a new issue with a clear title and description
- Include steps to reproduce the bug
- Add screenshots if applicable

### Suggesting Features

- Open an issue with the `enhancement` label
- Describe the feature and its use case
- Explain why it would benefit AMCAT aspirants

### Adding Problems

We welcome new AMCAT-style problems! To add a problem:

1. Create a new entry in `prisma/amcat-questions.ts`
2. Include:
   - Clear problem statement
   - Input/output format
   - Sample test cases (at least 2)
   - Hidden test cases (at least 5)
   - Appropriate difficulty and tags

### Code Contributions

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages: `git commit -m "feat: add feature description"`
6. Push and open a Pull Request

## Code Style

- Use TypeScript for all new files
- Follow existing code patterns
- Use meaningful variable and function names
- Add comments for complex logic

## Development Setup

```bash
npm install
npm run dev
```
